# 글씨인아트센터 웹 업데이트 계획서

> 작성 2026-06-12. 현재 사이트(Next.js 16 + Supabase)의 심층 분석과 우선순위 로드맵.
> 목표: ①예술 공방다운 디자인 완성 ②수업·상담 신청을 웹에서 완결 ③상품 판매·갤러리로 확장.

## A. 심층 분석 — 현재 상태

### A-1. 디자인 — "두 개의 디자인 시스템" 문제 (가장 중요)

**핵심 발견: 디자인 정의가 둘로 갈라져 있다.**

| | Vault `06_Design/DESIGN.md` | 실제 웹 `globals.css` |
|---|---|---|
| 골격 | 모노크롬(흑백) 코어 | 먹(ink)·한지(paper) 톤 |
| 강조색 | **인주색 #B23A30** (도장 붉은색, 페이지당 한 번) | 강조색 없음 (회색조만) |
| 리듬 | 큰 파스텔 색블록 섹션(lime·lilac·cream·mint…) | 단조로운 한지 배경 |
| 폰트 | 가변 산세리프 미세 무게(320~700) | Noto Serif/Sans KR + Pretendard |

→ 마케팅 카드(파이프라인)는 DESIGN.md를 따르고, 웹은 다른 길을 갔다. **둘 중 하나로 통일해야 브랜드가 한 목소리를 낸다.** 권장: 웹을 DESIGN.md 쪽으로 수렴 — 특히 **인주색 강조 도입**(지금 웹엔 브랜드 강조색이 없어 밋밋함)과 **파스텔 색블록 섹션 리듬**. 예술 공방의 사이트가 회색조만으로는 "절제"가 아니라 "심심"으로 읽힌다. 인주색 한 점이 글씨 브랜드의 정체성이다.

**디자인 강점**: 8px 그리드, 타이포 스케일, 자간·줄간격 토큰이 체계적으로 정의돼 있음. framer-motion으로 모션도 준비됨. 토대는 좋다 — 색의 서사가 빠졌을 뿐.

### A-2. 콘텐츠 — 정보 정합성

- **대표 정보**: DirectorProfile에 KCDA 교육위원·서예술대전 초대작가·예술대전 심사위원 등 수록 — Vault `대표_프로필.md`와 대체로 일치. 단 Vault엔 수상 30회·전시 17회 상세가 있는데 웹은 요약본. **전시·수상 타임라인을 살리면 작가 신뢰도가 시각적으로 강해진다**(예술 사이트의 핵심 자산).
- **수업 정보 불일치 3건** (정합성 점검 필요):
  1. 시간표 요일: 웹 `data.ts`는 "화요일" 포함, Vault `수업안내_종합.md`는 "월·화·일 정규 수업 없음". → 한쪽이 틀림. **확인 필요.**
  2. 시간 표기: 웹은 "3:00~5:00"(12시간제 혼용), Vault는 "15:00~17:00"(24시간제). → 24시간제로 통일 권장.
  3. 가격: 웹 메타데이터엔 "원데이클래스 50,000원"이 있으나 `data.ts` 본문엔 가격이 없음. → 가격 정보를 데이터에 명시.
- **클래스 종류**: 붓 캘리·붓펜 캘리·펜드로잉·서예·원데이 — Vault와 일치. 좋음.

### A-3. 신청·상담 — "측정 불가능한 깔때기" (사업적으로 가장 아까운 부분)

현재 모든 CTA가 **외부로 빠져나간다**: ContactModal·CTASection 모두 `tel:010-2497-4310` + 카카오 채널(`pf.kakao.com/_xkETdn`)로만 연결. 회원/게시판 시스템(Supabase auth + RLS)은 잘 갖춰져 있는데 **정작 수업·상담 신청을 받는 폼이 없다.**

문제: ①전화·카톡은 전환을 측정할 수 없다 → 마케팅 KPI("문의 유입")가 영원히 추측으로 남는다 ②영업시간 외 문의를 놓친다 ③신청자 정보가 구조화되지 않아 `02_Students/` 데이터로 연결이 안 된다. **이게 마케팅 시스템 전체와 웹을 잇는 끊어진 고리다.**

### A-4. 보안·기술 (이전 점검 재확인)

- 🔴 `save-logo` API 인증 없음 + admin 3개 페이지(gallery·schedule·texts) 인증 가드 없음 → 본 계획 P0에 포함.
- 🟡 이미지 무거움(g2.jpeg 3.7MB) + public/ 중복 저장(g1~g7 두 벌) → 예술 사이트인데 갤러리 로딩이 느리면 치명적.
- 🟢 RLS·env 관리·robots는 양호.

## B. 업데이트 로드맵 (우선순위)

### P0 — 보안·정합성 (즉시, 0.5~1일)

1. admin 하위 3페이지에 `getUser` + `is_admin` 가드 추가 (또는 middleware에서 `/admin/*` 일괄 보호).
2. `save-logo` 라우트 삭제(일회성 도구) — 남긴다면 admin 인증 + Storage 전환.
3. 수업 정보 3건 불일치 확정·통일 (요일·시간제·가격). **운영자 확인 필요 항목.**

### P1 — 신청·상담 폼 (핵심, 2~3일)

마케팅 시스템과 웹을 잇는 단일 최대 가치 작업.

- Supabase에 `inquiries` 테이블 + RLS(누구나 insert, admin만 select). 필드: 이름·연락처·관심 클래스·희망 시간대·메시지·유입경로(인스타/블로그/검색/지인)·생성일시.
- `/class`·CTASection·ContactModal에 **신청 폼** 추가 (전화·카톡은 보조로 유지).
- admin에 문의 목록 페이지 + 상태(신규/연락완료/등록) 관리.
- **유입경로 필드가 KPI를 실측으로 바꾼다** — Vault 성과로그의 "문의 유입"이 추측에서 데이터로.
- (선택) 신규 문의 시 카카오/이메일 알림.

### P2 — 디자인 통일 (예술성, 3~4일)

- `globals.css`에 인주색(#B23A30) 강조 토큰 도입 + 핵심 CTA·도장 모티프에 희소 적용.
- DESIGN.md의 파스텔 색블록 섹션 리듬을 롱폼 페이지(about·class)에 도입 — 섹션마다 색이 바뀌는 에디토리얼 호흡.
- 대표 페이지에 전시 17회·수상 30회 타임라인 컴포넌트 (Vault `대표_프로필.md`가 데이터 소스).
- 이미지 최적화: WebP 변환 + `<Image>` 적용 + 중복 제거 (용량 1/5~1/10).
- DESIGN.md ↔ globals.css를 **단일 토큰 소스로 동기화** (장기적으로 색·타이포 정의가 두 곳에서 갈라지지 않게).

### P3 — 갤러리 강화 (확장 1, 2~3일)

- 기존 gallery 테이블 활용. 작품 상세(제목·기법·연도·크기·설명) + 카테고리 필터(캘리/펜드로잉/서예).
- 작가 작품과 수강생 작품 구분 전시 → 수강생 작품은 "나도 할 수 있다"는 사회적 증거(퍼소나 전략과 연결).
- 고해상 이미지 + 라이트박스. 예술 사이트의 얼굴.

### P4 — 상품 판매 (확장 2, 5~7일)

- schema에 이미 `gallery_sale_fields` 마이그레이션 흔적 있음 → 판매 기능 일부 설계됨. 점검 후 확장.
- 상품(작품·굿즈·부재료) + 장바구니 + 주문. 결제는 **토스페이먼츠/포트원** 등 국내 PG 연동 (직접 결제 구현 금지 — 보안·정산).
- 재고·주문 관리 admin. 배송 정보.
- 주의: 결제·정산은 사업자 정보·전자상거래법 고지(반품·교환) 필수 — 법적 검토 후.

## C. 마케팅 시스템과의 연결 (이 웹의 진짜 가치)

1. **모든 채널 CTA를 웹 신청 폼으로** — 인스타·블로그 카피의 CTA를 전화번호 대신 사이트로. 파이프라인 `CTA_BY_INTENT`를 "수강 문의하기" → 웹 신청 링크로 교체.
2. **유입경로 → 성과로그 자동 연결** — 신청 폼의 유입경로 데이터가 `03_Agent_System/성과로그/`의 attribution을 채운다. 수동 입력 마찰 제거.
3. **수강생 데이터 → `02_Students/`** — 신청·등록 데이터가 빈 정원 표의 실데이터 소스.
4. **갤러리 작품 → 인스타 카드 소재** — 웹 갤러리 이미지를 마케팅 카드 생성의 자산으로 재활용.

→ 웹이 마케팅 깔때기의 **도착점(전환 측정)이자 출발점(콘텐츠 소재)**이 된다. 지금은 끊어져 있다.
