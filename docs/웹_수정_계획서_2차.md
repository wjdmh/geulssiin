### 공통 SEO 문제 (전 페이지)
1. **google-site-verification이 placeholder** ("GOOGLE_VERIFICATION_CODE_HERE") — 서치콘솔 미등록 상태. 등록하면 네이버·구글 유입 측정 시작.

## B. 디자인 — DESIGN.md 대조 결과와 결론

**현황**: 두 디자인 언어가 공존한다.
- Vault DESIGN.md: 모노크롬 코어 + 파스텔 색블록 + 인주색 한 점 + 알약 버튼 (Figma 마케팅 캔버스 기반 — 인스타 카드가 사용 중)
- 실제 웹: 먹(ink)/한지(paper) 웜 뉴트럴 + Noto Serif 세리프 + 직각 버튼 — **그 자체로 완성도 있게 통일된 에디토리얼 언어**

**결론 (1차 계획서에서 수정)**: 웹을 DESIGN.md로 갈아엎는 것은 비추천. 웹의 먹/한지·세리프는 서예 공방에 더 어울린다. 대신 **"공유 토큰 정렬"**:

1. **인주색(#B23A30)을 웹의 공식 강조 토큰으로 채택** — 현재 /contact 버튼에 하드코딩됨 → `globals.css`에 `--seal` 토큰 추가, "페이지당 한 점" 원칙(DESIGN.md 철학)으로 핵심 CTA·도장 모티프에만.
2. **DESIGN.md를 2부 구성으로 개정** — §마케팅 캔버스(카드·배너: 현행 유지) + §웹(먹/한지·세리프·직각 버튼·인주 한 점 명문화). 두 매체가 "인주색·서여기인·절제"라는 같은 정체성을 다른 옷으로 입는 구조로 단일 진실화.
3. **색블록 리듬 선택 실험** — About 스토리 섹션 1곳에 cream/lilac 색블록을 시험 적용해보고 어울리면 확장, 아니면 폐기.

## C. 관리자 UX — "편하게 올리고, 한눈에 본다"

### 현황 진단 (admin 코드 분석)
- **admin 홈**: 메뉴 카드 6개뿐 — 숫자가 없다. 신규 문의가 왔는지, 갤러리가 몇 점인지 들어가보기 전엔 모름.
- **갤러리 업로드**: Storage 직접 업로드는 잘 구현됨. 그러나 **1장씩만** 업로드 가능 + 필드 8개(제목·설명·카테고리·가격·기법·크기·연도·판매)를 매번 입력 → 작품 10점 올리려면 같은 과정 10번. 공방에서 폰으로 올리기엔 부담.
- **알림 부재**: 신규 수업 신청이 와도 아무도 모름 — admin에 들어가야 보임.

### 개선 계획

**C-1. admin 대시보드 홈 개편 (반나절, 효과 최대)**
메뉴 카드 위에 요약 줄 추가: 🔴 신규 문의 N건(최근 3건 미리보기) · 갤러리 N점 · 이번 주 수업 일정 · 회원 N명. 신규 문의 카드에 빨간 배지.

**C-2. 갤러리 다중 업로드 (1일)**
드래그&드롭으로 여러 장 선택 → 썸네일 목록에서 제목만 빠르게 입력(나머지 필드는 선택, 나중에 수정 버튼으로 보완) → 일괄 등록. "일단 올리고 나중에 다듬는" 흐름으로 전환. 모바일에서 사진 여러 장 선택 동작 확인.

**C-3. 신규 문의 알림 (반나절, 기존 인프라 활용)**
헤르메스 크론 활용: 매일 아침 inquiries에서 status=new 확인 → 디스코드로 "신규 신청 N건: 이름·관심클래스" 브리핑. 별도 이메일 인프라 불필요 — 이미 가진 에이전트가 알림벨이 된다.

**C-4. 시간표 관리 보완 (소)**
admin/schedule에 "확정 운영 요일은 수목금토" 안내 표시 + 화요일 잔존 행 정리 안내.
